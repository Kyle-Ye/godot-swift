Protocol

Godot.NativeScript

A nativescript type.

Declaration

protocol NativeScript:Godot.AnyNativeScript

Overview

Any Swift type (class, struct, enum) can be used as a Godot nativescript by conforming to this protocol.

At a minimum, a conforming type must support an init(delegate:) initializer, which will be called when this nativescript is instantiated, and attached to an instance of its associated Delegate type.

You can register signals, properties, and methods to make them accessible from GDScript, by specifying a nativescript Interface. The Interface type is its own result builder, which provides a convenient syntax for specifying nativescript interfaces:

@Interface 
static 
var interface:Interface 
{
    Interface.signals 
    {
        ExampleSignal.self 
    }
    Interface.methods 
    {
        exampleMethod(delegate:foo:) <- "example_method"
    }
    Interface.properties 
    {
        \.exampleProperty            <- "example_property"
    }
}

The first parameter of any exported nativescript instance method must be labeled delegate: and have the static type Delegate. As its name suggests, this parameter provides the nativescriptâ€™s means of interacting with the game engine.

Warning

Nativescript delegates are only guaranteed to be alive for the duration of the instance method call. A Swift nativescript may outlive, or become detached from its original delegate. This is especially important for unmanaged delegate types, such as Unmanaged.Node.

Conforming types must be registered in the interface of Godot.Library, otherwise, Godot Swift will be unable to synthesize its remaining requirements.
